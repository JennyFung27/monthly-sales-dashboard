# monthly-sales-dashboard

## Project Overview

This project presents an interactive monthly sales performance dashboard designed to monitor revenue, customer activity, average order value, regional performance, and product-level trends.

The dashboard consolidates multiple business metrics into a centralized reporting view, helping stakeholders quickly understand overall performance, identify growth drivers, investigate underperforming areas, and support recurring business reviews.

## Business Objective

The main objective of this project was to create a self-service reporting dashboard that allows users to:

- Monitor monthly revenue and customer performance
- Compare month-over-month changes
- Evaluate regional and province-level performance
- Identify product-level revenue contribution
- Track changes in average order value
- Analyze daily sales trends
- Support recurring management reporting and business planning

## Tools Used

- Microsoft Excel
- Power Query
- PivotTables
- PivotCharts
- Conditional Formatting
- Excel formulas and calculated fields

## Key Metrics

The dashboard tracks the following business metrics:

- Revenue
- Customer Count
- Average Order Value
- Month-over-Month Change
- Revenue Share
- Customer Share
- Regional Performance
- Product Performance
- Daily Sales Trend

## Dashboard Features

### Interactive Filters

Users can filter the dashboard by:

- Month
- Region

These filters allow stakeholders to review performance across different periods and geographic markets.

### Executive KPI Summary

The top section of the dashboard provides a high-level view of:

- Total revenue
- Total customer count
- Average order value
- Month-over-month performance

### Regional Performance Analysis

The regional performance table compares:

### Product Performance Analysis 

The product-level section evaluates:

- Revenue by product term
- Revenue share

### Daily Sales Trend

The daily trend chart shows changes in sales activity throughout the month and helps identify:

- Sales peaks
- Low-performance periods
- Daily fluctuations
- Product-level contribution over time

## Data Preparation

The reporting process included:

1. Importing raw sales data into Excel
2. Cleaning and transforming the data using Power Query
3. Standardizing date, region, product, revenue, and customer fields
4. Creating calculated metrics for revenue, customer count, and average order value
5. Calculating month-over-month changes
6. Building PivotTables and PivotCharts
7. Adding slicers and conditional formatting for interactive analysis

## Analysis Performed

The dashboard was used to:

- Compare monthly sales performance
- Identify regional revenue and customer growth
- Evaluate whether revenue changes were driven by customer volume or average order value
- Analyze product contribution to total revenue
- Identify high-performing and underperforming markets
- Monitor daily sales patterns
- Support recurring performance discussions


## Business Value

This dashboard improves visibility into sales performance by consolidating multiple KPIs into a single reporting view.

It supports faster decision-making by helping stakeholders:

- Identify revenue drivers
- Detect performance gaps
- Compare regional and product-level results
- Monitor recurring trends
- Prepare for monthly business reviews
